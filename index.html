Intro to Data Mesh

The concept of data mesh was first shared by Zhamak Dehghani of ThoughtWorks (LinkedIn; Twitter) as a reaction to seeing large customers spending far more on big data platforms but failing to see value from the investment. For her, centralized data infrastructure, especially things like a single data lake owned and managed by a specialized data infrastructure team, created complexity without value for most companies. Data producers (services) treating data as a by-product instead of a product.

The high-level data mesh concept is organizational alignment around creating data products, where ownership of data products extends beyond getting data into a centralized data warehouse/lake, but through to actual consumption by analysts and data scientists. It aligns data engineering with data production from services AND data analysis.

Data mesh has 4 pillars:

Domain-oriented, decentralised data ownership and architecture - no more data ownership by a data warehouse/lake team, give ownership to those who know the data best (domain driven design but for data) and can package it best for others.

Creating data products with true data owners - I feel this one is often phrased think of your data as a product but not all data should be productized. Data product owners need to have SLAs/SLOs/SLIs for their data products, just like any other product owner would.

A self-service data infrastructure platform for creating data products, serving data products to consumers, and consumption of data products - often referred to as the overall data platform consisting of one to many data nodes.

Federated computational governance, which is important but I feel should be left for last - basically, keep decision making as locally as possible but have some centralized control so everyone doesn't have to reinvent the data access security rules.

Zhamak has been, for good reason, reticent to make default architectural recommendations. A data mesh needs a strong technological basis but organizational buy-in and structure is far more important than any one technology or product. If you do not have true data product ownership, no piece of software will make your data mesh drive the value you expect from it.
